# `mocida-rs` — bindings, not a reimplementation

The Rust side is **FFI bindings + a safe wrapper** over the C toolkit. The C
library does all rendering and event handling. **Never reimplement widget
logic in Rust.** "Add a port" / "add coverage" means *add more wrappers*, never
rewrite behavior.

## Layout

- **`mocida-sys`** — raw `extern "C"` bindings generated by `bindgen` over
  `wrapper.h`. C enums become newtype tuple structs (e.g.
  `sys::UIWrapMode(pub i32)`). Stays buildable without SDL on disk via a stub
  header tree.
- **`mocida`** — the safe, idiomatic wrapper. Every public header
  `mocida/src/headers/uikit/*.h` maps to one module under `mocida/src/`.

## Conventions to follow when adding wrappers

- **Ownership via a `moved` flag.** A widget owns its `*mut UI*` and frees it
  in `Drop`. After `into_widget()` / `into_raw()` transfers ownership to a
  parent, set the flag so `Drop` does **not** double-free.
- **Callbacks via trampolines.** Box the closure, pass `Box::as_ref(&state)` as
  the `void* userdata`, recover it in an `extern "C"` trampoline. Store the box
  on the wrapper so its address stays stable. If ownership was transferred,
  `std::mem::forget` the box instead of dropping it (avoids a dangling
  `userdata`).
- **Chainable builders**: setters take `self` and return `Self`.
- **Newtype enums**: construct the bindgen newtype when calling FFI, e.g.
  `sys::UIFillMode(mode as i32)`.
- **Re-export** new public types from `mocida/src/lib.rs` so `mocida::Foo`
  resolves (and README examples compile).
- `mocida_alloc.h` and `perf.h` are **macro-only** and intentionally unwrapped.

## Verifying changes

`cargo check`/`build` (via `python build.py rust`) type-checks your calls
against the generated bindings. If you add/rename C functions, the bindgen
output only refreshes when `wrapper.h` or the build script change — force a
clean rebuild (`python build.py rust --force`) so new symbols appear.
