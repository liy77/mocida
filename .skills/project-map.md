# Project map

A monorepo with two packages plus a cross-platform build layer.

```
/                      monorepo root
├── build.py           build orchestrator (Turbo-style task graph)
├── setup.py           — (no; setup lives under mocida/, see below)
├── release.py         release orchestrator (menu/CLI + CHANGELOG.md)
├── build.ps1 / build.sh   thin wrappers that forward to build.py
├── sync-from-windows.sh   WSL helper: mirror source into a Linux checkout
├── CHANGELOG.md       prepended by release.py
├── mocida/            the C UI toolkit (Windows-first, on SDL3)
│   ├── build.py setup.py release.py docs.py   package scripts (Python)
│   ├── src/headers/uikit/*.h   public C API (one header per feature)
│   ├── src/*.c                 implementation
│   └── (vendored at setup time) vcpkg/ mimalloc/ SDL/ SDL_image/ SDL_ttf/ .webview2/
└── mocida-rs/         Rust bindings (Cargo workspace)
    ├── mocida-sys/    raw FFI generated by bindgen over wrapper.h
    └── mocida/        safe, idiomatic wrapper (one module per C header)
```

## Hard facts to remember

- **All build scripts are Python 3** (`build.py`, `setup.py`, `release.py`,
  `docs.py`). The old `.bat` / `.sh` / `.ps1` build scripts were removed —
  **do not recreate them.** The only shell files left are thin wrappers
  (`build.ps1`, `build.sh`) and `sync-from-windows.sh`.
- **Vendored dependencies live UNDER `mocida/`**, not at the repo root:
  `mocida/vcpkg`, `mocida/mimalloc`, `mocida/SDL`, `mocida/SDL_image`,
  `mocida/SDL_ttf`, `mocida/.webview2`. `mocida/setup.py` clones/bootstraps
  them there. If you ever find them at the repo root, that is the stale old
  layout — they belong under `mocida/`.
- **Build output** goes to `mocida/build/<platform>/<config>` where platform
  is `win32` / `linux` / `darwin`. The static lib is `mocida.lib` (Windows)
  or `libmocida.a` (Unix).
- **The Rust side links the C lib** via env vars `MOCIDA_INCLUDE_DIR`
  (`mocida/src/headers`) and `MOCIDA_LIB_DIR` (the build output dir). `build.py`
  sets these automatically for the `rust` task.
- Scripts are **stdlib-only Python** — do not add pip dependencies to them.
